EMBEDDED DATA FIELD NAMING CONVENTION
======================================================

Pattern:  C{set}_{proposal}_{attribute}
          C{set}_{proposal}_{attribute}_label

Examples:
  C1_P1_Country       = code shown to respondent in Set 1, Proposal 1, Country attribute
  C1_P1_Country_label = display text shown to respondent (EN or PL depending on language)
  C1_P2_UnitType      = code for Set 1, Proposal 2, Unit Type
  C10_P2_Environment  = code for Set 10, Proposal 2, Environment attribute

Sets:      1 through 10
Proposals: P1, P2
Attributes: Country, Personnel, UnitType, Proximity, Economic, Noise, Environment

Total code fields:  140  (10 sets x 2 proposals x 7 attributes)
Total label fields: 140  (same, with _label suffix)
AttrOrder:            1  (pipe-delimited string recording row order shown to respondent)
                    ---
Total embedded data: 281 fields

======================================================
RESPONSE VARIABLE NAMING CONVENTION
======================================================

Pattern:  QC{set}Q{question}

QC{set}Q1 = Which proposal did the respondent prefer?
             1 = Prefer Proposal 1
             2 = Prefer Proposal 2
             3 = Accept either
             4 = Reject both

QC{set}Q2 = Rating of Proposal 1 (1=Strongly Oppose, 5=Strongly Support)
QC{set}Q3 = Rating of Proposal 2 (1=Strongly Oppose, 5=Strongly Support)

Example: QC3Q1, QC3Q2, QC3Q3 = responses to conjoint Set 3
